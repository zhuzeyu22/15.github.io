关于多维数组的构造方法

fi15
2017-11-03 
大家都知道 如何直接初始化一个数组。

一维数组 char a[n];
二维数组 char a[n][m];
三维数组 char a[n][m][p];

因为数组是需要在编译时确定大小的，并且存放在栈中，或者静态区。

那麽我们如何获得 动态数组 ？


大家也都知道 数组就是一片连续的内存空间。

那麽，任意大小、任意维度的数组，也不过是一块和大小和维度相关的内存空间罢了。

所有动态申请的空间 无非就是 malloc 、 new ，在堆中开辟空间，然后获得一个返回的指针。

malloc(n); 
malloc(n * m); 
malloc(n * m * p);

但是我们无法获得数组名、无法进行下标运算获得我们想要的数组空间。

因为指针本身只是代表一个地址，没有多余的类型信息，所以我们需要对指针进行类型转换，才能够随心所欲的使用。


所以不同类型的指针，开始起作用了，int * 、 char * 、 struct *  ... ... function (*)();
当然也包含 char (*)[] ;

char (*a)[n]; 
char (*a)[n][m]; 
char (*a)[n][m][p]; 

这些指向一个数组的指针 (*a) 便成为了我们所想要的数组名，(*a)[x][x][x] 可以进行下标运算。
于是指针获得了进化，进化出了数组的长度信息、大小信息、类型信息。
